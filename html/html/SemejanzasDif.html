Entre XML y JSON

Semejanzas entre XML y JSON



Se puede utilizar una biblioteca serializadora XML para convertir un documento JSON en un archivo XML, de la misma manera que podría escribir una página HTML utilizando la etiqueta o crear imágenes.
Ambos formatos tienen estructuras jerárquicas en las que cada campo tiene su nombre y valor separados por comas.
JSON y XML pueden utilizarse para la serialización de datos (convertir una estructura de datos a un formato para su almacenamiento o transmisión).
JSON y XML son excelentes formas de almacenar datos en un servidor web.
Los formatos autodescriptivos se diseñaron para que fueran legibles para el ser humano y legibles para la máquina.
Tanto JSON como XML tienen un buen soporte para las definiciones de contenido y la validación.

Diferencias entre XML y JSON

XML JSON
XML tiene etiquetas de inicio y finalización. JSON no contiene etiquetas de inicio y finalización.
Soporte XML para Comentarios. JSON no admite Comentarios.
Los datos XML se pueden cambiar a otros formatos como texto sin formato. Los datos JSON no se pueden cambiar a otros formatos.
Mientras que XML se ha ampliado desde SGML. JSON se amplía desde JavaScript.
La sintaxis de XML es mas compleja. La sintaxis de JSON es más ligera.
XML es mejor para la configuración. JSON es mejor para los servicios web.
